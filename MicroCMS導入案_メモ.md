# MicroCMS導入検討メモ

## 1. 概要
Second Memory 公式サイト（静的HTMLサイト）におけるブログ・コラム更新を効率化するため、ヘッドレスCMSである「MicroCMS」を導入する。

## 2. なぜMicroCMSか？
*   **運用コストの削減**: HTMLを直接編集したり、GitHubへの直接プッシュ（ビルド工程の意識）をせずに、スマホやPCの管理画面から記事を投稿できる。
*   **デザインの自由度**: 外部ブログ（Note等）へ飛ばすのではなく、現在の美しいデザインを維持したまま記事だけを差し替えられる。
*   **日本語対応**: 管理画面が完全に日本語であり、直感的に操作できる。
*   **無料枠の活用**: 個人・小規模サイトであれば無料プランで十分に運用可能。

## 3. 導入のステップ
1.  **MicroCMSアカウント作成**: サービスへの登録とサービス情報の初期設定。
2.  **APIスキーマ定義**: ブログ記事として必要な項目（タイトル、本文、日付、画像、カテゴリ等）を定義する。
3.  **HTMLの書き換え**:
    *   `index.html`（トップページ）: 最新記事3〜4件をAPI経由で取得し、動的に描画するスクリプトを追加。
    *   `article_template.html`: 指定された記事IDの内容を自動で表示する「汎用テンプレート」を作成。
4.  **Netlify連携（Webhook）**: MicroCMSで「公開」ボタンを押すと、Netlifyが自動的に最新状態でサイトを更新（再ビルド）するように設定する。

## 4. 期待される効果
*   「思い立った時にすぐに記事を書ける」環境の構築。
*   エンジニアリング（コード編集）とコンテンツ作成の分離によるスピード向上。
*   サイト内SEOの強化（記事が増えるほどサイトの評価が高まる）。
