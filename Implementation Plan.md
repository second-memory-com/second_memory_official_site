# Second Memory 公式サイト 設計図

「Second Memory」公式サイトの立ち上げに向けた、初期段階の設計図です。

## プロジェクトの目的
「Second Memory」というプロダクトまたはブランドの公式ウェブサイトを構築し、その価値や情報を効果的に伝えます。

## 決定事項（要件定義: コーポレート・自然調和路線へ修正）
以下の内容で「Second Memory株式会社」公式サイトの構築を進めます。
- **コンセプト**: 「身体の記憶（Second Memory）から、脳を調律する」認知メカニズムからヒントを得たコンディショニング・システム（ソフトウェア・映像コンテンツ）
- **ターゲット**: 思考のノイズ（情報の波による疲れ、ブレインフォグ、集中力のムラ）に悩み、本来のポテンシャルを取り戻したい方
- **表現方針**: 医療的・ネガティブな表現は控え、「本来の自分」「ノイズのない状態」といったポジティブな表現へ昇華する。
- **デザイン**: 「先端テクノロジー（サイバー）」ではなく、「自然の中での深呼吸」「霧が晴れてフォーカスが合う感覚」を表現する、ナチュラルで透明感のあるデザイン。激しいスポーツ感ではなく、静かで研ぎ澄まされたコンディショニングを演出。
- **主要コンテンツ**: コーポレートサイトとしての信頼感を持たせる構成（Mission, Service, Company, Contact等）。

## 提案する構成（暫定）

### 技術スタック
モダンで高品質なデザインを実現するため、以下の構成を提案します。
- **フロントエンド**: HTML5, Vanilla CSS（※初心者が理解しやすいようJSや複雑な装飾は極力排除したシンプル設計）
- **デザイン**: プレミアムで動きのある、レスポンシブなデザイン
- **ホスティング（将来案）**: Netlify, Vercel 等

### セキュリティ・ガイドライン（初心者向け安全設計）
初めてサイトをネットに公開する際でも、安全に運用するための絶対のルールです。

1. **APIキーやパスワードの直書き禁止（最重要）**
   - 外部サービス（Google Maps, メール送信ツール, 予約システムなど）と連携するための「APIキー」や「パスワード」は、**絶対に `index.html` や `main.js` の中に直接書いてはいけません。**
   - HTML等のファイルに書かれた内容は、ブラウザで「ソースを表示」するだけで世界中の誰でも見ることができてしまうためです。
   - どうしても必要な場合は、ホスティングサービス（NetlifyやVercel等）の「環境変数設定」などの安全な裏側の仕組みを使います。
2. **外部スクリプトの安易な読み込み禁止**
   - `<script src="...">` でよくわからない外部サイトのコードを読み込まないようにします。公式で安全性が確認されているもの（Google Fontsなど）のみを使用します。
3. **フォームのスパム対策**
   - お問い合わせフォームなどを自作する際は、そのまま設置するとスパムメールの標的になります。Google Formsや、セキュリティ対策が施された専用のフォームサービス（Netlify Forms, Formspreeなど）を利用します。

### サイト構造（コーポレート版構造案）
「Second Memory 株式会社」の公式サイトとしての信頼感と、コンディショニング・システムの魅力を両立させる構成です。

- [ ] **1. Hero**: 「霧が晴れ、本来の自分（ポテンシャル）を取り戻す」自然で透明感のあるビジュアルとメッセージ。
- [ ] **2. Mission / Vision**: なぜSecond Memory株式会社が存在するのか、目指す世界（脳のハンドルを自分の手に取り戻す）。
- [ ] **3. Product (Self-Driver)**: 認知メカニズムにアプローチする4つのコア・トレーニング（バランス、指の巧緻性、高速音読、逆暗唱）。
- [ ] **4. Company**: 会社概要（Second Memory株式会社について）。
- [ ] **5. Contact / Booking**: 問い合わせ案内。
